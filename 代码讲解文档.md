## 一、整体架构

整个程序由 6 个 Python 模块组成，解决以下科学问题：

> **已知**：在多个发动机工作点 $(π_c, T_3)$ 下，测量到带噪声的比推力 $F_{obs}$ 和比油耗 $SFC_{obs}$。
> **求解**：压气机等熵效率 $\eta_c$ 和涡轮等熵效率 $\eta_t$ 的后验概率分布。

数学框架为：

$$P(\theta | D) \propto P(D | \theta) \cdot P(\theta)$$

其中 $\theta = [\eta_c, \eta_t]$，$D$ 为全体观测数据。

**模块层次：**

```
main.py                        ← 总调度，串联所有步骤
├── turbojet_engine.py         ← 正向物理模型 y = f(θ, x)
├── virtual_data.py            ← 生成带噪声的"虚拟试验数据"
├── bayesian_model.py          ← 构建似然、先验、后验
├── mcmc_sampler.py            ← MCMC 采样求解后验分布
└── visualization.py           ← 全套诊断图与结果图
```

## 二、`turbojet_engine.py` — 发动机热力循环模型

### 2.1 物理背景

涡喷发动机将空气吸入后依次经过 5 个热力站位：

```
大气 (0) → 进气道 → 压气机 (1→2) → 燃烧室 (2→3) → 涡轮 (3→4) → 喷管 (4→5) → 排气
```

每个站位用总温 $T_{0x}$ 和总压 $P_{0x}$ 描述热力状态。

### 2.2 常数与假设

| 符号 | 数值 | 含义 |
|------|------|------|
| $T_0$ | 288.15 K | ISA 海平面大气温度 |
| $P_0$ | 101325 Pa | ISA 海平面大气压力 |
| $\gamma$ | 1.4 | 空气比热比 |
| $C_p$ | 1005 J/(kg·K) | 定压比热容 |
| $H_u$ | 43 MJ/kg | 航空煤油低热值 |
| $V_0$ | 0 m/s | 地面静止（飞行速度为零） |

假设：进气道无损失；燃烧室等压燃烧无压力损失；喷管完全膨胀（出口静压 = 大气压）；不计燃油质量流量（$1+f \approx 1$）；无机械损失与引气。

### 2.3 `compute_performance()` — 核心热力循环

这是整个程序的**物理基础**，计算给定 $(\pi_c, T_3, \eta_c, \eta_t)$ 下的发动机性能。

#### 步骤 1：进气道（站位 0 → 1）

地面静止工况，进气道无损失：

$$T_{01} = T_0 = 288.15\text{ K}, \quad P_{01} = P_0 = 101325\text{ Pa}$$

#### 步骤 2：压气机（站位 1 → 2）

压气机将空气从低压绝热压缩到高压，存在等熵效率 $\eta_c$（真实过程比等熵过程耗功更多）。

**等熵压比温比**（理想压缩过程中温度与压比的关系）：

$$\tau_{c,s} = \pi_c^{\frac{\gamma-1}{\gamma}}$$

> 推导：绝热可逆过程 $T/T_0 = (P/P_0)^{(\gamma-1)/\gamma}$，代入压比 $\pi_c$ 即得。

**实际压气机出口总温**（考虑效率，真实温升大于等熵温升）：

$$T_{02} = T_{01} \left(1 + \frac{\tau_{c,s} - 1}{\eta_c}\right)$$

> 说明：$\eta_c = \frac{T_{02,s} - T_{01}}{T_{02} - T_{01}}$（等熵温升 / 实际温升），变形后即上式。

**压气机出口总压**：

$$P_{02} = P_{01} \cdot \pi_c$$

**压气机单位质量耗功**（推动每千克气体所需的功）：

$$W_c = C_p (T_{02} - T_{01}) \quad [\text{J/kg}]$$

#### 步骤 3：燃烧室（站位 2 → 3）

等压加热，出口温度即为给定的涡轮入口温度 $T_3$：

$$T_{03} = T_3, \quad P_{03} = P_{02}$$

**燃油-空气比**（由能量守恒推导，燃料化学能转化为气体焓增）：

$$f = \frac{C_p(T_{03} - T_{02})}{H_u}$$

> 物理意义：$f$ 为每千克空气所需的燃油质量。$H_u$ 是燃料完全燃烧释放的热量。

代码对 $f$ 做有效性检查（$0 < f < 0.08$），确保处于物理合理范围。

#### 步骤 4：涡轮（站位 3 → 4）

涡轮驱动压气机，功率平衡（忽略机械损失）：

$$W_\text{turbine} = W_c$$

**涡轮实际出口总温**（从功率平衡直接得出）：

$$T_{04} = T_{03} - \frac{W_c}{C_p}$$

**计算涡轮膨胀比**：需先由效率定义推出等熵出口温度。

涡轮效率定义（实际温降 / 等熵温降）：

$$\eta_t = \frac{T_{03} - T_{04}}{T_{03} - T_{04,s}}$$

变形：等熵出口总温为

$$T_{04,s} = T_{03} - \frac{T_{03} - T_{04}}{\eta_t}$$

**涡轮膨胀比**（由等熵关系推导）：

$$\pi_t = \left(\frac{T_{03}}{T_{04,s}}\right)^{\frac{\gamma}{\gamma-1}}$$

**涡轮出口总压**：

$$P_{04} = \frac{P_{03}}{\pi_t}$$

#### 步骤 5：喷管（站位 4 → 5）

喷管将燃气完全等熵膨胀至大气压，将压力势能转化为动能。

**喷管出口静温**（等熵膨胀，出口静压 $P_5 = P_0$）：

$$T_5 = T_{04} \left(\frac{P_0}{P_{04}}\right)^{\frac{\gamma-1}{\gamma}}$$

**喷管出口速度**（能量守恒：焓降 = 动能增量）：

$$V_e = \sqrt{2 C_p (T_{04} - T_5)}$$

#### 步骤 6：性能参数计算

**比推力**（地面静止时，单位质量流量产生的推力）：

$$F_{sp} = V_e - V_0 = V_e \quad [\text{N/(kg/s)}]$$

> 推力公式：$F = \dot{m}(V_e - V_0) + (P_5 - P_0)A_e$，完全膨胀时 $P_5 = P_0$，地面静止时 $V_0 = 0$，故 $F_{sp} = V_e$。

**比油耗**（单位推力的燃油消耗率）：

$$SFC = \frac{f}{F_{sp}} \quad [\text{kg/(N·s)}]$$

### 2.4 `compute_batch()` — 批量计算

对 $N$ 个工作点循环调用 `compute_performance()`，返回数组形式，用于贝叶斯采样中对所有工作点同时评估。

### 2.5 `cycle_states()` — 全站位热力参数

返回包含所有站位 $T$、$P$ 的字典，供调试和可视化使用，计算逻辑与 `compute_performance()` 相同。

## 三、`virtual_data.py` — 虚拟试验数据生成

### 3.1 设计思路

模拟 Mathcad 计算流程：先通过性能图选出合理的工作点 $(\pi_c, T_3)$，再在这些工作点处用"真实"效率计算理论性能，最后叠加测量噪声，得到虚拟试验数据。

### 3.2 `plot_performance_maps()` — 性能参数图

**计算内容**：遍历 $\pi_c \in [4, 14]$（步长 200 个均匀点）和 5 条 $T_3$ 曲线（1100、1200、1300、1400、1500 K），调用 `compute_performance()` 计算每个工作点的 $F_{sp}$ 和 $SFC$，绘制两张曲线图。

**图像解读**：
- 比推力图：$T_3$ 越高，比推力越大；$\pi_c$ 存在最优值（太低压缩不足，太高膨胀比不够）。
- 比油耗图：$\pi_c$ 存在最小耗油率点（热力学最优增压比）。
- **用途**：通过对比两图，选出同时满足"比推力大、耗油率低"的 $(\pi_c, T_3)$ 组合作为工作点。

### 3.3 `plot_optimal_region()` — 最优工作区域云图

**计算内容**：在 $\pi_c \times T_3$ 的二维网格（80×60 共 4800 个点）上计算性能，用 `contourf` 填充色图，直观显示参数空间中的性能分布。

### 3.4 工作点选取

```python
OPERATING_POINTS = [
    (6,1200), (8,1200), (10,1200),
    (6,1300), (8,1300), (10,1300),
    (7,1350), (9,1350), (11,1350),
    (6,1400), (8,1400), (10,1400),
    (7,1450), (9,1450), (11,1450),
]  # 共 15 个工作点
```

选取依据：$T_3 \in [1200, 1450]$ K（高推力区），$\pi_c \in [6, 11]$（较优耗油率区），5 种温度 × 3 种增压比的正交设计，保证对 $\eta_c$ 和 $\eta_t$ 的充分辨识。

### 3.5 `generate_virtual_data()` — 核心数据生成

**第一步：计算无噪声理论性能**

用已知的"真实值" $\eta_c^{true} = 0.85$，$\eta_t^{true} = 0.88$（可配置）调用正向模型：

$$F_i^{true} = f(\pi_{c,i},\ T_{3,i},\ \eta_c^{true},\ \eta_t^{true})$$

**第二步：确定噪声标准差**

噪声水平设为均值的 1%（可配置）：

$$\sigma_F = \text{noise\_pct} \times \overline{F^{true}}$$
$$\sigma_{SFC} = \text{noise\_pct} \times \overline{SFC^{true}}$$

> 使用均值而非逐点标准差，使噪声均匀（homoscedastic），符合贝叶斯模型的等方差假设。

**第三步：叠加高斯噪声**

$$F_i^{obs} = F_i^{true} + \varepsilon_F^{(i)}, \quad \varepsilon_F^{(i)} \sim \mathcal{N}(0,\ \sigma_F^2)$$
$$SFC_i^{obs} = SFC_i^{true} + \varepsilon_{SFC}^{(i)}, \quad \varepsilon_{SFC}^{(i)} \sim \mathcal{N}(0,\ \sigma_{SFC}^2)$$

使用 `np.random.default_rng(seed)` 保证可重复性。

**返回数据字典**包含：`pi_c`、`T3`、`F_obs`、`SFC_obs`（含噪声的"观测值"）、`F_true`、`SFC_true`（无噪声真实值）、`sigma_F`、`sigma_SFC`（噪声标准差）、`eta_c_true`、`eta_t_true`、`N`。

## 四、`bayesian_model.py` — 贝叶斯反演模型

本模块严格对应"一、参数反演问题的数学描述"的第三至第八节。

### 4.1 误差模型（第三节）

假设测量误差服从零均值高斯分布，且各工作点误差相互独立：

$$\varepsilon_F \sim \mathcal{N}(0,\ \sigma_F^2), \quad \varepsilon_{SFC} \sim \mathcal{N}(0,\ \sigma_{SFC}^2)$$

### 4.2 `log_likelihood()` — 对数似然函数（第四至第七节）

**单点联合似然**（第五节，$F$ 与 $SFC$ 误差独立，联合概率为乘积）：

$$P(F_i^{obs},\ SFC_i^{obs} | \theta) = \frac{1}{2\pi\sigma_F\sigma_{SFC}} \exp\left[-\frac{(F_i^{obs} - F_i^{model})^2}{2\sigma_F^2} - \frac{(SFC_i^{obs} - SFC_i^{model})^2}{2\sigma_{SFC}^2}\right]$$

**多组独立观测的联合似然**（第六节，N 组数据相互独立，联合概率为连乘）：

$$P(D | \theta) = \prod_{i=1}^{N} P(F_i^{obs},\ SFC_i^{obs} | \theta)$$

**取对数后（第七节）**，乘积变为求和，常数项（$-N\ln(2\pi\sigma_F\sigma_{SFC})$）对优化无影响可忽略：

$$\ln P(D | \theta) \propto -\sum_{i=1}^{N} \left[\frac{(F_i^{obs} - F_i^{model})^2}{2\sigma_F^2} + \frac{(SFC_i^{obs} - SFC_i^{model})^2}{2\sigma_{SFC}^2}\right]$$

**代码实现**：

```python
inv_2sigF2   = 1.0 / (2.0 * sigma_F ** 2)    # 预计算常数，避免循环内除法
inv_2sigSFC2 = 1.0 / (2.0 * sigma_SFC ** 2)

log_L = 0.0
for i in range(N):
    F_model, SFC_model, ok = engine.compute_performance(...)
    if not ok:
        return -np.inf                         # 物理无效 → 概率为 0 → 对数为 -∞
    log_L -= (F_obs[i] - F_model)**2 * inv_2sigF2
    log_L -= (SFC_obs[i] - SFC_model)**2 * inv_2sigSFC2
```

> **数学意义**：对数似然实质是加权最小二乘损失的负值。$\ln P(D|\theta)$ 越大，意味着模型预测值与观测值越接近，$\theta$ 越合理。当某工作点热力计算物理不合理（如涡轮温度为负）时，返回 $-\infty$ 表示该参数组合被完全拒绝。

### 4.3 `log_prior()` — 均匀先验（第八节）

先验表达工程知识：压气机和涡轮效率的合理范围为 $[0.60, 0.98]$（基于工程实践）。

$$P(\theta) = \begin{cases} \frac{1}{(0.98-0.60)^2} & \text{若 } \eta_c, \eta_t \in [0.60, 0.98] \\ 0 & \text{其他} \end{cases}$$

取对数：在范围内返回 $0$（常数的对数），范围外返回 $-\infty$。均匀先验不影响后验形状，仅起到边界约束作用。

### 4.4 `log_prior_gaussian()` — 高斯先验（可选）

当有工程经验值时，用高斯分布作为先验（对效率有大致估计但不确定）：

$$\ln P(\eta_c) = -\frac{(\eta_c - \mu_c)^2}{2\sigma_c^2}, \quad \ln P(\eta_t) = -\frac{(\eta_t - \mu_t)^2}{2\sigma_t^2}$$

两参数独立，联合先验对数为两者之和，同时加入硬边界约束（$0.5 < \eta < 1.0$）。

### 4.5 `log_posterior()` — 对数后验（第八节）

贝叶斯公式取对数：

$$\ln P(\theta | D) = \ln P(D | \theta) + \ln P(\theta) + \text{const}$$

代码实现：先计算先验（若在范围外立即返回 $-\infty$，跳过耗时的似然计算），再加上似然：

```python
lp = log_prior(theta, ...)          # 均匀先验
if not np.isfinite(lp):
    return -np.inf                  # 先验为零，直接拒绝
ll = log_likelihood(theta, ...)     # 似然计算（耗时）
return lp + ll
```

### 4.6 `compute_likelihood_surface()` — 似然曲面

在 $\eta_c \times \eta_t$ 的 70×70 网格上遍历计算 `log_likelihood()`，返回网格矩阵用于可视化。主要用于**直观展示反问题的可辨识性**：若似然曲面有清晰的峰值，说明数据能唯一确定效率参数。

## 五、`mcmc_sampler.py` — MCMC 采样器

### 5.1 为何使用 MCMC

后验分布 $P(\theta|D)$ 没有解析形式（正向模型是复杂的热力方程），无法直接积分。MCMC 通过构造一条以后验为稳态分布的马尔可夫链，用链的统计特性近似后验分布。

### 5.2 `MHSampler` — 自适应 Metropolis-Hastings 算法

#### 算法流程

**初始化**：设置初始参数 $\theta^{(0)}$，计算 $\ln P(\theta^{(0)}|D)$。

**每步迭代** $t = 1, 2, \ldots, N_{total}$：

**第 1 步：提案（Proposal）**

从以当前点为中心的二维高斯分布抽取候选点：

$$\theta^* \sim q(\theta^* | \theta^{(t-1)}) = \mathcal{N}\left(\theta^{(t-1)},\ \text{step\_size}^2 \cdot I_2\right)$$

代码：`proposal = all_samples[t-1] + rng.normal(0.0, step_size, dim)`

**第 2 步：接受概率**

对称提案下（$q(\theta^*|\theta) = q(\theta|\theta^*)$），Metropolis-Hastings 接受率为：

$$\alpha = \min\left(1,\ \frac{P(\theta^*|D)}{P(\theta^{(t-1)}|D)}\right) = \min\left(1,\ e^{\ln P(\theta^*|D) - \ln P(\theta^{(t-1)}|D)}\right)$$

代码在对数域计算：`log_alpha = log_p_prop - log_p_curr`（避免指数运算溢出）

**第 3 步：接受/拒绝**

抽取 $u \sim \text{Uniform}[0,1]$：

$$\theta^{(t)} = \begin{cases} \theta^* & \text{若 } \ln u < \log\alpha \text{（接受）} \\ \theta^{(t-1)} & \text{否则（拒绝，链留原位）} \end{cases}$$

代码：`if np.log(rng.uniform()) < log_alpha: accept`

> **直觉理解**：如果候选点的后验概率更高，一定接受；如果更低，以一定概率接受（避免陷入局部极值）。

#### 自适应步长调整

目标接受率 0.234（理论最优，对二维参数空间 $d=2$，最优接受率约为 $0.234$）。

每 `adapt_interval=200` 步计算当前接受率：
- 若接受率 **低于** 目标 → 步长 **缩小**（提案分布太宽，跳的太远总被拒绝）
- 若接受率 **高于** 目标 → 步长 **放大**（提案分布太窄，每步移动太小，探索效率低）

```python
def _adapt_step(step, acc_rate, target=0.234, scale=0.9):
    if acc_rate < target:
        return step * scale      # 缩小步长
    return step / scale          # 放大步长
```

自适应只在预热期（前 `adapt_burnin` 步）内进行，之后步长固定，确保链的平稳性。

#### 预热期（Burn-in）

前 `n_burnin` 个样本被丢弃，原因：
1. 初始点 $\theta^{(0)}$ 是人为设定，不是从后验中取样；
2. 预热期链正在"找到"后验主体区域；
3. 丢弃预热期样本后，剩余样本近似服从目标后验分布。

### 5.3 `MultiChainMH` — 多链并行采样

从先验范围内随机选取 `n_chains` 个（默认 4 个）互相远离的初始点，各自独立运行 MH 采样链。多链的优势：
- 提高覆盖后验的概率（单链可能陷入局部区域）；
- 提供收敛诊断（Gelman-Rubin 统计量）。

### 5.4 `_gelman_rubin()` — Gelman-Rubin 收敛诊断

$\hat{R}$ 统计量比较**链间方差**与**链内方差**，判断各链是否已收敛到同一分布。

设 $M$ 条链，每条链有 $N$ 个样本，第 $j$ 维参数的 $\hat{R}$ 计算如下：

**链内方差 $W$**（各链自身的方差均值，反映链内探索程度）：

$$W = \frac{1}{M}\sum_{m=1}^{M} s_m^2, \quad s_m^2 = \frac{1}{N-1}\sum_{t=1}^{N}(\psi_m^{(t)} - \bar{\psi}_m)^2$$

**链间方差 $B$**（各链均值之间的差异，反映链间探索差异）：

$$B = \frac{N}{M-1}\sum_{m=1}^{M}(\bar{\psi}_m - \bar{\bar{\psi}})^2$$

**后验方差估计**（$W$ 与 $B$ 的加权平均）：

$$\hat{V} = \frac{N-1}{N}W + \frac{B}{N}$$

**$\hat{R}$ 统计量**：

$$\hat{R} = \sqrt{\frac{\hat{V}}{W}}$$

**判断准则**：$\hat{R} < 1.1$ 表示链已充分收敛（链间差异可忽略）。若 $\hat{R}$ 远大于 1，说明各链尚未混合，需增加采样步数。

## 六、`visualization.py` — 结果可视化

### 6.1 `plot_likelihood_surface()` — 对数似然曲面图

### 6.2 `plot_mcmc_diagnostics()` — MCMC 诊断图（3×2 子图）

诊断目的：确认链已收敛、样本质量良好。

**链图（Trace Plot）**：绘制 $\eta_c$ 和 $\eta_t$ 随迭代步数的变化。
- 理想状态：预热期后链在真实值附近上下随机游走，不存在明显漂移。
- 红色虚线标记预热结束点，橙色横线为真实值，紫色虚线为后验均值。

**自相关函数（ACF）**：

$$\rho(k) = \frac{\text{Cov}(x_t, x_{t+k})}{\text{Var}(x_t)} = \frac{\overline{(x_t - \bar{x})(x_{t+k} - \bar{x})}}{\overline{(x_t - \bar{x})^2}}$$

- ACF 越快衰减到 0，说明样本间自相关性越低，有效独立样本数越多。
- 红色虚线为 95% 置信区间边界（$\pm 1.96/\sqrt{N}$），超出边界说明存在显著自相关。

**累积均值收敛图**：

$$\bar{\theta}_n = \frac{1}{n}\sum_{t=1}^{n}\theta^{(t)}$$

后验均值的累积估计随样本数增加应逐渐稳定（收敛），蓝色阴影为 $\pm 2\sigma/\sqrt{n}$ 的不确定性区间。

### 6.3 `plot_posterior_marginals()` — 后验边缘分布图

对每个参数独立分析后验分布（边缘化掉另一个参数）。

**直方图**：将后验样本分 60 个区间统计频率，归一化为概率密度。

**核密度估计（KDE）**：用 Silverman 带宽的高斯核平滑后验样本，得到连续的概率密度曲线：

$$\hat{p}(\eta) = \frac{1}{Nh}\sum_{i=1}^{N}K\!\left(\frac{\eta - \eta^{(i)}}{h}\right), \quad K(u) = \frac{1}{\sqrt{2\pi}}e^{-u^2/2}$$

**关键统计量**标注：
- 红色实线：真实值（仅在虚拟数据问题中可知）；
- 橙色虚线：后验均值 $\hat{\eta} = \mathbb{E}[\eta | D]$；
- 蓝色阴影：95% 贝叶斯可信区间（Credible Interval），由样本的 2.5% 和 97.5% 分位数定义；
- 文本框：相对误差 $\frac{|\hat{\eta} - \eta^{true}|}{\eta^{true}} \times 100\%$ 和后验标准差。

### 6.4 `plot_posterior_contour()` — 二维联合后验等高线图

将 $(\eta_c, \eta_t)$ 的二维后验联合分布可视化，揭示两参数的相关性。

**二维 KDE**（Scott 带宽）：

$$\hat{p}(\eta_c, \eta_t) = \frac{1}{N}\sum_{i=1}^{N}K\!\left(\frac{\eta_c - \eta_c^{(i)}}{h_c},\ \frac{\eta_t - \eta_t^{(i)}}{h_t}\right)$$

图中展示：蓝色填充等高线（浓度越深概率密度越高）；稀疏散点（从后验样本随机抽取 800 个）；真实值（红色星形）；后验均值（橙色三角形）；先验均匀分布范围（灰色虚线方框）。

### 6.5 `plot_model_fit()` — 后验预测对比图（PPD）

**后验预测分布（Posterior Predictive Distribution）**：用后验样本中的 300 个随机参数点各自计算预测性能，得到预测的不确定性区间（PPD）。

**模型拟合效果评估**：绘制"观测值 vs. 预测值"散点图，理想情况下所有点落在 $y=x$ 对角线（红色虚线）上。

**拟合优度 $R^2$**：

$$R^2 = 1 - \frac{\sum_i (y_i^{obs} - y_i^{pred})^2}{\sum_i (y_i^{obs} - \bar{y}^{obs})^2}$$

$R^2$ 越接近 1，说明模型解释的方差比例越大，拟合越好。

**竖向误差棒**：每个观测点对应的 PPD 95% 区间，反映后验参数不确定性传播到预测的程度。

### 6.6 `plot_residuals()` — 残差分析图（2×2 子图）

验证误差正态性假设（贝叶斯模型的基础假设之一）。

**标准化残差**（消除量纲）：

$$r_i^F = \frac{F_i^{obs} - F_i^{model}}{\sigma_F}, \quad r_i^{SFC} = \frac{SFC_i^{obs} - SFC_i^{model}}{\sigma_{SFC}}$$

若假设成立，$r_i \sim \mathcal{N}(0, 1)$，绝大多数点应在 $[-2, +2]$ 内。

**Q-Q 正态图（Quantile-Quantile Plot）**：将样本分位数与理论正态分位数对比。若点落在对角直线上，说明残差近似正态分布，误差假设合理。
